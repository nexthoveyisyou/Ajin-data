## ===========================================
##          회원관리 프로그램 설계
## ===========================================

## -------------------------
## 1. 프로그램 개요
## -------------------------
- 회원의 정보 등록, 조회, 수정, 삭제, 목록 기능 제공하는 간단한 회원관리 프로그램
- 파이썬 기초 문법, 조건문, 반복문, 함수, 리스트/딕셔너리, 파일 입출력 등을 활용

## -------------------------
## 2. 개발 목표
## -------------------------
① 회원 정보를 저장할 수 있다.
② 저장된 회원 정보를 목록으로 볼 수 있다.
③ 특정 회원을 검색할 수 있다.
④ 회원 정보를 수정할 수 있다.
⑤ 회원 정보를 삭제할 수 있다.
⑥ 프로그램 종료 후에도 데이터가 남도록 파일에 저장할 수 있다.

## -------------------------
## 3. 회원 정보 항목 설계
## -------------------------
- 회원 1명당 저장할 정보는 다음과 같이 구성한다.
    -> 회원번호
    -> 이름
    -> 전화번호
    -> 이메일
    -> 가입일


## -------------------------
## 4. 프로그램 기능 구성
## -------------------------
    4-1. 회원 등록 기능
        - 새로운 회원의 정보를 입력받아 저장한다.

        - 입력 예시: 이름, 나이, 전화번호, 이메일
        - 처리 내용:
            -> 회원번호는 자동 증가 방식으로 부여
            -> 입력된 회원 정보를 리스트에 추가
            -> 저장 완료 메시지 출력

    4-2. 회원 전체 조회 기능
        - 현재 등록된 모든 회원 정보를 출력한다.

        - 처리 내용:
            -> 회원이 한 명도 없으면 “등록된 회원이 없습니다” 출력
            -> 회원이 있으면 회원번호 순서대로 출력

        - 출력 예시: 회원번호 / 이름 / 전화번호 / 이메일


    4-3. 회원 검색 기능
        - 이름 또는 회원번호로 특정 회원을 찾아 출력한다.

        - 처리 내용:
            -> 이름으로 검색 가능
            -> 회원번호로 검색 가능
            -> 검색 결과가 없으면 안내 메시지 출력


    4-4. 회원 정보 수정 기능
        - 기존 회원의 정보를 수정한다.

        - 처리 내용:
                -> 수정할 회원번호 입력
                -> 해당 회원이 존재하는지 확인
                -> 이름, 전화번호, 이메일 등의 정보를 새 값으로 변경
                -> 수정 완료 메시지 출력

    4-5. 회원 삭제 기능
        - 선택한 회원 정보를 삭제한다.

        - 처리 내용:
            -> 삭제할 회원번호 입력
            -> 회원 존재 여부 확인
            -> 삭제 확인 후 제거
            -> 삭제 완료 메시지 출력


    4-6. 파일 저장 기능
        - 회원 정보를 파일에 저장한다.

        - 처리 내용:
            -> 프로그램 종료 시 자동 저장 가능
            -> 또는 메뉴에서 수동 저장 가능
            -> txt로 저장 가능

    4-7. 파일 불러오기 기능
        - 프로그램 시작 시 저장된 회원 정보를 불러온다.

        - 처리 내용:
            -> 기존 파일이 있으면 읽어서 리스트에 저장
            -> 파일이 없으면 새로 시작
            -> 에러가 나지 않도록 예외 처리 필요


## -------------------------
## 5. 프로그램 실행 흐름
## -------------------------
- 프로그램은 메뉴 기반으로 동작한다.

- 메인 메뉴 예시
    1. 회원 등록
    2. 회원 전체 조회
    3. 회원 검색
    4. 회원 수정
    5. 회원 삭제
    6. 파일 저장
    7. 프로그램 종료

- 사용자는 번호를 입력하여 원하는 기능을 선택한다.

- 프로그램 흐름 예시:
    ① 프로그램 시작.          => 회원관리 프로그램 
    ② 저장된 회원 정보 불러오기 => 파일에 저장했다면 
    ③ 메뉴 출력              _ _
    ④ 사용자 선택 입력           |
    ⑤ 선택한 기능 실행       무한 반복 
    ⑥ 다시 메뉴로 돌아감      _ _|
    ⑦ 종료 선택 시 저장 후 종료

## -------------------------
## 6. 데이터 구조 설계
## -------------------------
- 리스트 안에 딕셔너리 형태가 가장 이해하기 쉽다.

- 이 구조를 사용하는 이유는 다음과 같다.
    -> 여러 회원 정보를 한 번에 저장하기 쉽다.
    -> 각 회원의 항목을 이름으로 구분할 수 있다.
    -> 수정/삭제/검색 기능 구현이 비교적 쉽다.


## -------------------------
## 7. 필요한 주요 함수 설계
## -------------------------
- 프로그램은 함수 단위로 나누어서 작성하는 것이 좋다.

- 함수 예시
    1) 메뉴 출력 함수
       메뉴를 화면에 출력한다.

    2) 회원 등록 함수
       사용자 입력을 받아 새 회원 정보를 생성한다.
       회원번호를 자동 부여한다.
       리스트에 추가한다.

    3) 회원 전체 조회 함수
       전체 회원 목록을 반복문으로 출력한다.

    4) 회원 검색 함수
       이름 또는 회원번호를 입력받아 일치하는 회원을 찾는다.

    5) 회원 수정 함수
       회원번호를 기준으로 회원을 찾고 정보를 수정한다.

    6) 회원 삭제 함수
       회원번호를 기준으로 회원을 찾아 삭제한다.

    7) 파일 저장 함수
       현재 회원 리스트를 파일에 저장한다.

    8) 파일 불러오기 함수
       파일에서 회원 정보를 읽어와 리스트에 저장한다.


## -------------------------
## 8. 예외 상황 설계
## -------------------------
- 프로그램을 만들 때는 정상 입력만 생각하지 말고 잘못된 입력도 고려해야 한다.

- 고려할 예외 상황
    -> 메뉴 번호가 잘못 입력된 경우
    -> 숫자를 입력해야 하는데 문자를 입력한 경우
    -> 존재하지 않는 회원번호를 입력한 경우
    -> 회원이 없는 상태에서 조회/수정/삭제를 시도한 경우
    -> 저장 파일이 없는데 불러오기를 시도한 경우

- 예외 처리 방향
    -> try-except를 사용하여 프로그램이 강제 종료되지 않게 한다.
    -> 잘못된 입력 시 다시 입력받도록 처리한다.
    -> 안내 메시지를 친절하게 출력한다.

- 예시:
    “숫자로 입력해주세요.”
    “해당 회원번호는 존재하지 않습니다.”
    “등록된 회원이 없습니다.”

## -------------------------
## 10. 개발 순서 제안
## -------------------------
- 1단계
    회원 정보를 리스트에 저장
    등록 / 조회 기능만 구현

- 2단계
    검색 / 수정 / 삭제 기능 추가

- 3단계
    함수로 나누어 코드 정리

- 4단계
    파일 저장 / 불러오기 추가

- 5단계
    예외 처리 추가

- 6단계
    코드 정리 및 주석 정리
